LLM Content Normalization - Before/After Examples

Purpose: Demonstrate how content normalization eliminates format bias in LLM evaluation exports.

Example 1: Markdown-Heavy Copy

❌ Before Normalization

Problem: Mixed markdown artifacts

**Revolutionize Your Workflow with AI** Transform your business with *cutting-edge* AI solutions. ### Key Benefits - **Automate** repetitive tasks - Make data-driven decisions - Scale efficiently **CTA:** Start your free trial today

Issues:

  • Markdown artifacts: **, *, -
  • Inconsistent list formatting
  • Mixed heading styles
  • UI label present (**CTA:**)

✅ After Normalization

Solution: Clean, consistent HTML

<p><strong>Revolutionize Your Workflow with AI</strong></p> <p>Transform your business with <em>cutting-edge</em> AI solutions.</p> <h3>Key Benefits</h3> <ul> <li><strong>Automate</strong> repetitive tasks</li> <li>Make data-driven decisions</li> <li>Scale efficiently</li> </ul> <p>Start your free trial today</p>

Results:

  • ✅ Clean HTML tags
  • ✅ Consistent structure
  • ✅ No markdown artifacts
  • ✅ UI labels removed

Example 2: Plain Text with UI Labels

❌ Before Normalization

Problem: UI labels and inconsistent structure

Hero: Revolutionize Your Workflow Section 1: Transform your business with AI Benefits: Automate repetitive tasks Make data-driven decisions Scale efficiently CTA: Start your free trial

Issues:

  • UI labels: Hero:, Section 1:, Benefits:, CTA:
  • No HTML structure
  • Plain text paragraphs
  • Unclear content hierarchy

✅ After Normalization

Solution: Labels removed, structure added

<p>Revolutionize Your Workflow</p> <p>Transform your business with AI</p> <p>Automate repetitive tasks Make data-driven decisions Scale efficiently</p> <p>Start your free trial</p>

Results:

  • ✅ All UI labels removed
  • ✅ Proper paragraphs
  • ✅ Clean content
  • ✅ Consistent formatting

Example 3: Structured Content Object

❌ Before Normalization

Problem: Inconsistent structured output

{ "headline": "Revolutionize Your Workflow", "sections": [ { "title": "Key Benefits", "content": "Transform your business.", "listItems": [ "Automate tasks", "Make decisions" ] } ] } // Exported as plain text with markdown: Revolutionize Your Workflow Key Benefits Transform your business. - Automate tasks - Make decisions

Issues:

  • Mixed format (object + markdown)
  • Inconsistent with other versions
  • List items as markdown

✅ After Normalization

Solution: Structured HTML hierarchy

<h1>Revolutionize Your Workflow</h1> <h2>Key Benefits</h2> <p>Transform your business.</p> <ul> <li>Automate tasks</li> <li>Make decisions</li> </ul>

Results:

  • ✅ Proper HTML hierarchy
  • ✅ Headline as H1
  • ✅ Section title as H2
  • ✅ Clean list structure

Actual Exported HTML Structure

This is exactly what appears in the hidden #llm-evaluation-data section:

<section id="llm-evaluation-data" data-purpose="llm-copy-extraction" data-evaluation-scope="copy-only" style="display: none;"> <!-- Original Copy --> <article data-copy-id="__original__" data-copy="original" data-copy-kind="original" data-copy-label="Original Copy"> <h2>Original Copy</h2> <div data-copy-body="true" data-copy-normalized="true"> <p>Transform your business with cutting-edge AI solutions.</p> <ul> <li>Automate repetitive tasks</li> <li>Make data-driven decisions</li> <li>Scale efficiently</li> </ul> </div> </article> <!-- Generated Copy 1 --> <article data-copy-id="abc-123" data-copy="generated-1" data-copy-kind="generated" data-copy-label="Generated Copy 1"> <h2>Generated Copy 1</h2> <div data-copy-body="true" data-copy-normalized="true"> <h1>Revolutionize Your Workflow with AI</h1> <p>Transform your business with cutting-edge AI solutions.</p> <h2>Key Benefits</h2> <ul> <li>Automate repetitive tasks</li> <li>Make data-driven decisions</li> <li>Scale efficiently</li> </ul> <p>Start your free trial today</p> </div> </article> <!-- Generated Copy 2 (Voice Applied) --> <article data-copy-id="def-456" data-copy="generated-2" data-copy-kind="voice" data-copy-label="CEO's Voice"> <h2>CEO's Voice</h2> <div data-copy-body="true" data-copy-normalized="true"> <p>Here's the truth: your competitors are already using AI.</p> <p>The question isn't whether you should adopt AI—it's how fast you can move.</p> <ul> <li>Automate in minutes, not months</li> <li>Scale without hiring</li> <li>Compete with enterprise-level efficiency</li> </ul> <p>Try it free. No credit card required.</p> </div> </article> </section>

Key Observations:

  • ✅ All versions use data-copy-normalized="true"
  • ✅ All use consistent HTML tags (<p>, <ul>, <li>)
  • ✅ No markdown artifacts remain
  • ✅ No UI labels present
  • ✅ Same structural patterns across versions
  • ✅ Only content differs, not format

Validation Checklist

Use this checklist to verify normalization is working correctly:

Check What to Look For Pass/Fail
1. Attribute Present All div[data-copy-body] have data-copy-normalized="true" ✅ Pass
2. No Markdown No **, *, or - at start of lines ✅ Pass
3. No UI Labels No Hero:, CTA:, Section:, etc. ✅ Pass
4. Clean HTML Tags Only <h1-3>, <p>, <ul>, <li>, <strong>, <em> ✅ Pass
5. Consistent Structure All versions with similar content use same tag hierarchy ✅ Pass
6. Normalized Spacing Max 2 consecutive newlines, no excessive whitespace ✅ Pass
7. HTML Escaped Special characters (<, >, &) are escaped ✅ Pass
8. Meaning Preserved Original content meaning unchanged, only format improved ✅ Pass

Why This Matters for LLM Evaluation

❌ Without Normalization

External LLM sees:

  • Version 1: Clean professional copy
  • Version 2: Markdown artifacts everywhere
  • Version 3: UI labels mixed in

Result: Version 1 scores higher because it looks cleaner, not because it's better copy.

Bias: Format quality affects content score ❌

✅ With Normalization

External LLM sees:

  • Version 1: Clean HTML
  • Version 2: Clean HTML
  • Version 3: Clean HTML

Result: Scores based purely on content quality, message clarity, persuasiveness.

Fair: Only content matters, not format ✅

Bottom Line:

We're not improving the copy. We're removing evaluation noise.

This ensures fair, unbiased comparison of actual content quality.
