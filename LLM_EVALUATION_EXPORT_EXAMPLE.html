LLM Evaluation Export Structure - Example

This document shows the HTML structure added to exports for external LLM evaluation.

🔒 HIDDEN: Context Section

Location: After input summary, before prompt evaluation

Purpose: Provides configuration context for LLM evaluation

Visibility: Hidden with display: none;

<!-- LLM EVALUATION SECTION: Only use content inside #llm-evaluation-context and #llm-evaluation-data when independently evaluating copy outputs. Ignore all scoring and comparison UI elsewhere in this file. --> <section id="llm-evaluation-context" data-purpose="llm-context" style="display: none;"> <div data-context-field="language">English</div> <div data-context-field="tone">Professional</div> <div data-context-field="word-count">Medium: 100-200</div> <div data-context-field="target-audience">Business professionals</div> <div data-context-field="key-message">Increase productivity</div> <div data-context-field="call-to-action">Sign up for free trial</div> </section>
✅ Contains ONLY user input configuration
✅ NO scores, rankings, or evaluations
✅ Helps external LLMs understand the brief

🔒 HIDDEN: Copy Data Section

Location: After context section

Purpose: Provides raw copy versions without any scoring or UI

Visibility: Hidden with display: none;

✅ NO scores, winner labels, or rankings
✅ NO recommendations or explanations

Location: After hidden sections, before output cards

Purpose: Informs users about the structured data

Visibility: Visible to users in the export

✅ Visible to users in the export
✅ Enhanced with data attributes for targeting

📖 How External LLMs Use This

Example Prompt for ChatGPT/Claude:

I've uploaded an HTML export from CopyZap. Please analyze the copy versions. IMPORTANT INSTRUCTIONS: 1. Only read content from these sections: - #llm-evaluation-context (for configuration) - #llm-evaluation-data (for copy versions) 2. Ignore everything else, including: - Scores and rankings - Comparison tables - Winner labels and explanations - Performance metrics - UI elements 3. For each copy version: - Extract the data-copy-label to identify it - Read content from data-copy-body="true" - Evaluate based on the context provided 4. Provide your independent assessment of which version is best and why.

JavaScript Extraction Example:

// Extract evaluation context const contextSection = document.getElementById('llm-evaluation-context'); const context = {}; contextSection.querySelectorAll('[data-context-field]').forEach(field => { const key = field.getAttribute('data-context-field'); context[key] = field.textContent.trim(); }); // Extract copy versions const dataSection = document.getElementById('llm-evaluation-data'); const versions = []; dataSection.querySelectorAll('article[data-copy-id]').forEach(article => { versions.push({ id: article.getAttribute('data-copy-id'), kind: article.getAttribute('data-copy-kind'), label: article.getAttribute('data-copy-label'), content: article.querySelector('[data-copy-body]').textContent.trim() }); }); console.log('Context:', context); console.log('Versions:', versions); // Result: // Context: { language: "English", tone: "Professional", ... } // Versions: [ // { id: "__original__", kind: "original", label: "Original Copy", content: "..." }, // { id: "abc-123", kind: "generated", label: "Generated Copy 1", content: "..." }, // { id: "def-456", kind: "voice", label: "CEO's Voice", content: "..." } // ]

✅ Success Criteria Met

  • ✅ Hidden context section with configuration only
  • ✅ Hidden data section with raw copy versions only
  • ✅ Clear data attributes on all containers
  • ✅ HTML comment guidance for external LLMs
  • ✅ No scores/rankings in hidden sections
  • ✅ Dynamic generation for all output types
  • ✅ Visible user note about structured data
  • ✅ Clean content (markdown stripped)
  • ✅ Backward compatible with existing exports
  • ✅ Easy targeting with CSS selectors or JavaScript

🎯 Key Data Attributes Reference

Attribute Values Purpose
data-purpose llm-context, llm-copy-extraction Section type identifier
data-evaluation-scope copy-only Indicates no scoring data
data-copy-id UUID or __original__ Unique version identifier
data-copy-kind original, generated, voice, modified Type of copy version
data-copy-label User-friendly name Display name for version
data-copy-body true Marks actual copy content
data-context-field language, tone, word-count, etc. Configuration field name
