## STRATEGIC BRIEF (Before Copy)

The hero has one job: stop the person who has already tried ChatGPT, Jasper, or any AI writing tool and is quietly dissatisfied with why it still does not feel like a real system.

They do not think they need a new tool. They think they need a better prompt, or more time, or a different template. The headline must correct that belief in under five words. The subheadline must name the gap they have been living with. The CTA must make the first action feel like a result, not a commitment.

The visual must show something they have never seen in any other writing tool: a version being scored, ranked, and selected — not just displayed.

### RECOMMENDED PRIMARY HEADLINE:

> **Writing was never the bottleneck. Deciding was.**

**Why this one:**
It lands the problem before naming the product. Every AI writing tool user has felt this — the tool gave them options and they still had to guess. This headline names that experience precisely without jargon. It creates immediate tension. It demands the next sentence.

**Fallback if A/B testing:**
> **You have ten drafts. You still do not know which one is right.**

This variant is more visceral for users who have actively experienced the problem. It is slightly longer but more emotionally specific.

## 6. HERO VISUAL RECOMMENDATION

### What to Show:

**The comparison output state** — not the input form, not a blank editor, not a chat interface.

Specifically: the moment after generation, showing two or three copy versions side by side, each with a score visible (e.g., 84 / 100), and one version highlighted with a visual indicator — a badge, a border, a crown — indicating it is the declared winner.

---

### Exact UI State to Capture:

- **2–3 copy output cards** visible in the frame (partial third card acceptable for depth)
- **Score displayed prominently** on each card — a number out of 100 or a percentage
- **Winner indicator** on the highest-scoring card — must be visually distinct (a badge reading "Winner" or "Best Version" or a highlighted border)
- **Brief evidence snippet visible** below the winner indicator — one line of the rationale, e.g.: *"Scores highest on clarity and audience fit. Strongest opening hook of the three versions."*
- **Sub-score breakdown visible** (partially) — suggest showing 3–4 labeled sub-score bars or chips (e.g., Clarity: 91, Persuasion: 78, Trust: 84) on the winning card
- **Decision badge** if present — e.g., "Best Conversion" or "Highest Clarity"
- **Version labels** visible — "Version A", "Version B", "Version C" — or numbered

---

### What Must NOT Be Shown:

- **No input forms** — the hero visual is not about setup, it is about the output value
- **No empty states or blank cards** — must show populated, real-looking output
- **No chat interface, prompt box, or "Enter your prompt" field** — this communicates "another prompt tool"
- **No loading spinners** — this is the moment of delivered value, not in-progress
- **No generic text placeholders** (Lorem ipsum, "Your copy here")
- **No feature sidebar, nav, or settings panels** — keep the visual focused on the comparison output only
- **No overly complex UI** — the visual should communicate "structured and clear" not "overwhelming"

---

### Copy in the Visual:

Use realistic-looking marketing copy in the output cards. Suggested format:

**Version A (the winner):**
A 2–3 sentence block of ad copy or landing page copy for a recognizable use case (e.g., SaaS product, service business). Score shown: 87. Decision badge: "Best Conversion."

**Version B:**
Same format, different approach (more feature-focused, less benefit-focused). Score: 73. No badge.

**Version C (optional):**
Third approach, partially visible. Score: 68.

---

### Visual Direction:

- **Light mode preferred for hero** — high contrast, scores should read immediately
- **Score numbers should be the largest numerical element** on the visual
- **Winner card should have clear but understated visual treatment** — a subtle colored border (orange or dark) is sufficient; avoid anything that looks like confetti or gamification
- **The overall visual tone:** professional, structured, analytical — not playful, not editorial
- **If animated:** a single transition from "scoring in progress" to "scores revealed" — 1.5 seconds max, then hold on the final state

---

### What This Visual Communicates:

1. This tool produces multiple versions (not just one)
2. Every version is evaluated (not just displayed)
3. A winner is identified with evidence (not left to the user to guess)
4. The interface is structured and professional (not a chat window)
5. The output is actionable (not just text on a screen)

This visual must make the hero visually distinct from every other AI writing tool screenshot in the market. Those visuals show text being generated into an editor. This visual shows decisions being made.

## NOTES FOR IMPLEMENTATION

**Headline font size:** Large enough that it reads in under one eye fixation — typically 52–64px desktop. This is not a moment for elegance — it is a moment for impact.

**Subheadline max width:** Cap at ~580px. Beyond that, line lengths become too long for comfortable reading at hero size.

**CTA button:** Primary CTA must be full-color, high contrast. If using orange brand color, white text on orange passes WCAG AA at typical button sizes. Do not use a gradient — it signals a different brand era.

**Secondary CTA:** Underline or arrow — no button border. The visual hierarchy must make the primary CTA the obvious first choice.

**Trust line:** 12–13px, gray. Below CTAs. Centered if CTAs are centered.

**Hero section whitespace:** Do not crowd this section. The visual and the copy need breathing room. The comparison visual should float — not be embedded in a crowded component.

**Mobile:** Stack headline → subheadline → primary CTA → secondary CTA → trust line vertically. Drop the hero visual below the copy block on mobile, or use a simplified single-card score view instead of the full comparison layout.
