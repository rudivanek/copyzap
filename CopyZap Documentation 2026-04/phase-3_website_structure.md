## 4. Conversion Strategy

---

### Primary CTA:
**"Start Free"** or **"Run Your First Comparison Free"**

The second variant is stronger — it names the differentiating action (comparison) and removes cost friction simultaneously. Test both.

**Where it appears:**
- Hero (primary button — highest visual priority)
- After the Decision Layer section (the highest-conviction moment on the page)
- After the Use Cases section (segment-specific readiness moment)
- Final CTA section (closing conversion)
- Features page footer
- How It Works page closing

---

### Secondary CTA:
**"See How It Works"** or **"Watch It Score"**

Serves visitors who are not ready to start but are curious. Captures them at the exploration stage and keeps them on the site rather than losing them to a bounce.

**Where it appears:**
- Hero (secondary link — below the primary button, lower visual weight)
- After the Problem section (before the solution is explained — for skeptical visitors who want to see before believing)
- Comparison section (for visitors asking "does this actually work?")

---

### When to Show Each:

| Moment | Visitor State | CTA to Show |
|---|---|---|
| Hero | First arrival — scanning | Primary CTA + Secondary CTA |
| Context break | Orienting | No CTA — let them read |
| Problem section | Recognizing pain | No CTA — let the problem land |
| Why existing tools fail | Frustration with current tools | Secondary CTA (see it work) |
| Solution explanation | Understanding the system | No CTA — earning the sale |
| How it works | Ready to understand effort | Secondary CTA ("see it live") |
| Decision layer | Highest conviction moment | Primary CTA |
| Use cases | Identified their role | Primary CTA (segment-aware if possible) |
| Features | Researching before committing | Primary CTA at section end |
| Comparison | Validating choice | Primary CTA |
| Proof | Trust-building | Primary CTA if strong proof / no CTA if proof is thin |
| Final CTA | Decision moment | Primary CTA — maximum size, maximum clarity |

---

### Friction Points:

1. **"What is the scoring based on?"** — Visitors may not trust algorithmic scoring without understanding the criteria. Friction: skepticism about whether scores are meaningful.

2. **"Is this just another AI writing tool with a score bolted on?"** — The category differentiation may not land if the visitor skims. Friction: dismissal before full understanding.

3. **"I already have Jasper / ChatGPT / a copywriter."** — The visitor has a current solution. The cost of switching feels unknown. Friction: inertia.

4. **"How much does it cost / will I run out of credits immediately?"** — Credit-based pricing creates anxiety about running up against limits before getting value. Friction: trial fear.

5. **"This looks complex."** — The depth of the feature set may signal that it is hard to use. Friction: perceived effort barrier.

6. **"I do not have time to learn a new tool."** — Common SaaS objection. Friction: time investment uncertainty.

---

### How to Reduce Friction:

**Friction 1 (Score credibility):**
- Show a real scored output with the evidence excerpt visible in the Decision Layer section
- Name all scoring dimensions explicitly — do not hide the methodology
- Include "What scores measure" explanation in the Help page and link to it from the scoring UI

**Friction 2 (Category confusion):**
- Lead with the category contrast in Section 2 before explaining CopyZap
- The comparison matrix must be clear and specific — not vague capability claims
- The hero visual must show the scoring/comparison interface, not a generic text editor

**Friction 3 (Existing solution inertia):**
- Name specific tools in the comparison section — do not be generic
- Frame the comparison as "the layer they are missing" not "we are better"
- The "Why existing tools fail" section must be specific about what the tools do NOT do — not general claims

**Friction 4 (Credit anxiety):**
- State free tier limits explicitly on the pricing page — no ambiguity
- Add a "What you can do for free" section that names concrete use cases
- In the final CTA, state the no-credit-card policy if applicable

**Friction 5 (Complexity perception):**
- Lead the How It Works page with "5 steps, under 60 seconds for a first output"
- Show the Quick Wizard path — the simplest input experience — as the default introduction
- Expert mode is presented as opt-in, not the default

**Friction 6 (Time investment):**
- The "first session" walkthrough on the How It Works page shows the complete process is short
- Emphasize that smart defaults mean minimal setup required
- "Your brief is the only thing you bring — CopyZap handles the rest"

## STRUCTURAL RULES FOR IMPLEMENTATION

1. **Section order is load-bearing.** The persuasion sequence is not decorative — it mirrors how conviction actually builds. Do not reorder sections without tracking conversion impact.

2. **The Decision Layer section (Section 7) must receive the strongest visual weight on the page** — after the Hero. It is the differentiator that no competitor can show. If it is visually weak or poorly explained, the entire positioning collapses.

3. **Features come last.** Section 10. Any visitor who is ready to convert earlier has a CTA available. Visitors who need features before converting will scroll. Do not put features higher — it makes CopyZap look like every other tool.

4. **No feature names before Section 5.** Before the system view is established, feature names are meaningless. "Deep Analysis" means nothing to a visitor who does not yet understand that evaluation is the problem being solved.

5. **Social proof placement is conditional.** If strong proof exists, place it between the Decision Layer and the Use Cases sections (maximum conviction moment). If proof is placeholder, put it before the final CTA only — do not let weak proof undermine the stronger sections above it.

6. **Every section must earn the next.** Read each section transition: does the end of the section make the reader want to see the next one? If not, the section is not doing its job.
